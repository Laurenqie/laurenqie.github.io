Hexo使用手册(1) 使用Github Pages搭建Hexo博客

2016/08/31 posted in  正经

为了安装Hexo，你需要拥有Git Command Line tools、Github账号、NVM和Node.js.

安装Hexo

git

Mac用户可以考虑使用Github Desktop或者Xcode自带的Git。

nvm

通过

curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.31.6/install.sh | bash

或者

wget -qO- https://raw.githubusercontent.com/creationix/nvm/v0.31.6/install.sh | bash

来安装NVM。

安装完毕以后运行

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh"

直接开始运行NVM。

用NVM部署Node.js

在终端运行

nvm install node

好了，现在可以装Hexo了！

Hexo

输入

npm install -g hexo-cli

大功告成！

部署Hexo到Github上

为 Hexo 安装 Git 插件

npm install hexo-deployer-git --save

config.yml改为

deploy:
  type: git
  repo: <repository url>
  branch: [branch]
  message: [message]

添加SSH key

添加key之后，github才知道修改文件的人是你哦。

ssh-keygen -t rsa -C "youremail@example.com"

终端会显示在你的用户主目录下增加了.ssh目录和id_rsa,id_rsa.pub这两个文件。

用cd命令跳转到.ssh目录以后ls列出目录下文件，

more id_rsa.pub

即可看到pub文件中内容，将内容复制到

GitHub-->“Account settings”-->“SSH Keys”-->Add a SSH key。

新建一个Github repository作为你的博客存放地

在Github新建repository，名称为

your-username.github.io

cd到你本地所创建的hexo文件夹，然后执行

$ git remote add origin git@github.com:你刚刚建好的repo

这样你的本地文件夹和就和Github上的repo建立起联系啦。

迁移你的MD文档

如果你之前的博客都是Markdown文章，将它们复制到

你的hexo文件夹/source/_post里即可。

注意hexo的frontmatter：
https://hexo.io/docs/front-matter.html

title: Hello World
date: 2013/7/13 20:46:25

push你的本地hexo文件夹到git上

hexo g
hexo d

或者

hexo g -d

在本地查看你的Hexo网站

hexo g
hexo s

或者

hexo g -s

一些注意事项

每次使用hexo之前要把NVM运行起来。

执行

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh"

即可。

本文的参考网址：

https://hexo.io/
